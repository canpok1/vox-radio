# 0052. コーナーに ID を導入し扱い回数・前回出演回番号を生成 LLM に渡す

- ステータス: 採用
- 日付: 2026-06-08

## コンテキスト

「新コーナーです」「久しぶりの復活コーナー」のような時間文脈を台本に出したいが、コーナーには回をまたぐ扱い回数情報がなく初回判定も復活判定もできない。キャストは出演回数を持つ（ADR-0037/0040）が、コーナーには非対称に存在しない。コーナーは Title で同定されるため（`CornerMap`・`manifest`・`cache`）、Title を変えると履歴照合が切れて回数集計が壊れる。回数を持たせる前提として Title 変更に耐える安定キーが要る。

## 決定

- **コーナー ID を必須化**: `config.CornerConfig` に `id` を追加し、同定キーを Title から ID へ統一する（`CornerMap`・`manifest.Build`・`cache` 集計）。check で未設定・番組内重複をエラーにする。
- **扱い回数を実績ベースで LLM へ**: `cache.CornerEntry` に id を保存し、過去エントリから「出演回数（今回含む・初回=1）」と「前回出演した回番号（初回=0）」を集計して `RundownCorner` に焼き込み、select / flow / write の 3 経路に渡す。LLM は `episode_number` との差で「N 回ぶり」を推論する。
- **境界変換はしない**: キャスト（ADR-0040）と異なり初回=1 のまま渡す。新規機能で既存入力との互換制約がなく、二重定義を避け直感性を優先する。
- **演出は情報提供のみ**: プロンプトに演出ルールは明記せず（ADR-0037 踏襲）、新フィールドの意味説明のみ追記する。
- **レガシー（id 無し）は集計対象外**（バックフィルせず 0 回=初回扱い。ADR-0037 踏襲）。

## 結果

実績ベースのため再生成でも扱い回数が変わらず、Title を変えても ID で履歴を追える。ID 統一で二重同定による混乱を避けられ、演出を強制せず自然な会話を阻害しない。トレードオフとして config への id 追記移行が必要で、同定キー移行が `CornerMap` の全呼び出し元に波及する。

## 検討した代替案

- **同定は Title 維持で集計だけ ID 化**: 二重同定が残り混乱するため却下。
- **ブランク回数をコードで算出して渡す**: 生値＋`episode_number` で LLM に推論させる方が単純で表現の自由度も高く却下。
- **キャスト同様に境界変換する**: 互換制約がなく不要、初回=1 の直感性を優先して却下。
