# 0045. direct に発音校正パスを追加し VOICEVOX のかな化取りこぼしによる誤読を抑制する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

VOICEVOX（OpenJTalk）の誤読が発生していた（例: 「頭突き」を「あたまつき」と発音）。音素レベルの誤読は漢字が残っている場合のみ起き、ADR-0027 で direct に集約したかな化（`line_conversions`）が正しく変換していれば発生しない。今回は direct の一発生成が「頭突き」を漢字のまま素通ししたのが直接原因。

かな化は LLM の文脈判断で行うため、誤読しやすい語の検出漏れ（素通し）が構造的に起こりうる。手動でのユーザー辞書登録は対象語が無数にあり非現実的で、検出漏れを救済する仕組みが必要になった。

## 決定

direct のかな化の後に「発音校正パス」を追加する。元台本とかな化結果を突き合わせ、未変換・誤読・意味不明な変換を検出して正しいかな表記で該当行を上書きする LLM 呼び出しを 1 回追加する。ADR-0027 で「コスト増のため却下」した別 LLM 呼び出しを、校正に限って品質目的で再導入する判断更新にあたる。

かな化の範囲は従来どおり「選択的かな化（誤読しやすい語のみかな化し漢字は残す）」を維持する。責務分担は既存 `line_conversions` と同じで、判断と正しいかな生成は校正 LLM、`(corner_index, line_index)` をキーにした上書き適用はプログラムが決定的に行う。不備検出時は自動修正のうえ修正件数・内容を stderr へ出力し、校正の欠落・失敗時は direct のかな化（さらに欠落時は元テキスト）へフェイルソフトでフォールバックする。

## 結果

- 素通しによる誤読を後段で救済でき、誤読率が下がる。漢字を残すためアクセント・イントネーションの自然さも維持される。
- 文脈依存の読みにも対応できる（静的辞書では不可能）。
- LLM 呼び出しが 1 回増えコスト・レイテンシが増すが、フェイルソフト設計のため校正失敗時も従来品質を保つ。
- LLM 出力を index 付き修正リストに限定するため、セリフの順序変更・行欠落/重複の事故をプログラム側で防げる。

## 検討した代替案

- **全文かな化**: 素通しを構造的に排除できるが、漢字がアクセント推定の手がかりのため同音語・複合語の抑揚が劣化しうる。誤読は校正パスで救済できるため不採用。
- **VOICEVOX ユーザー辞書の成長方式**: かな化と役割が重複し、静的辞書のため文脈依存の読みに対応できない。不採用。
- **形態素解析（kagome 等）で決定的にかな付与**: OpenJTalk と同系辞書で同様に誤読しうるうえ、アクセント情報を失い新規依存も増える。不採用。
