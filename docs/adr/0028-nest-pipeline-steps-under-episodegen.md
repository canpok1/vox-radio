# 0028. パイプライン各ステップを episodegen サブコマンド配下へ再編する（run のリネーム）

- ステータス: 採用
- 日付: 2026-06-03

## コンテキスト

CLI はフルパイプライン実行の `run` と、その構成ステップ（collect / rundown / script / synth / assemble / manifest）をすべてトップレベルに**並列配置**していた。このため「run の中で各ステップが順に走る」という包含関係がコマンド構成から読み取れず、どのコマンドがどこまで実行するのか理解しづらかった。

加えて `run` という名前は「どこまで実行するか」を表さない。実体は記事収集から台本生成・音声合成・組み立て・manifest 出力までを通して 1 本のエピソード（`episode.mp3`）を生成するオーケストレータである。

CLI 構造の決定は ADR-0009（cobra 採用）・ADR-0026（feedgen のサブコマンド化）に続く方針判断であり、前例に倣い記録する。

## 決定

- `run` を **`episodegen`** にリネームする（最終成果物=1 本のエピソードで命名。フォーマット非依存）。
- パイプライン 6 ステップを **`episodegen` 配下のサブコマンド**に再編する（例: `vox-radio episodegen collect`）。包含関係をコマンド階層で表現する。
- フルパイプラインは **`vox-radio episodegen`（引数なし）** で実行する。親コマンド自身に `RunE`（旧 run のロジック）を持たせ、`Args: cobra.NoArgs` で未知の positional 引数を弾く。
- **feedgen / init / config / profile はトップレベル据え置き**。feedgen は単一エピソードでなく複数エピソード横断の RSS 配信処理であり、エピソード生成パイプライン外のため。
- 後方互換エイリアス（旧 `run` / 旧トップレベル `collect` 等）は設けない。

## 結果

- run と各ステップの包含関係がコマンド階層で可視化され、実行範囲が直感的になる。
- 命名がフォーマット非依存になり、将来 mp3 以外の出力形式へ拡張しても陳腐化しない。
- トップレベルが init / episodegen / config / profile / feedgen に整理され、用途別の見通しが良くなる。
- トレードオフ: 既存の `vox-radio run` / `vox-radio collect` 等の呼び出しは破壊的に廃止される（エイリアスなし）。また episodegen は「フル実行の親」と「ステップ群のグループ」を兼ねる二役になる。

## 検討した代替案

- **`mp3gen` / `audiogen` へのリネーム**: 出力形式・媒体で命名する案。mp3gen は将来の形式変更で陳腐化、audiogen は clip の wav 素材と紛らわしく、かつ収集・台本生成まで担う実行範囲を過小に見せるため却下。
- **フラット構成のまま rename のみ**: 並列配置による包含関係の読みづらさが解消しないため却下。
- **`episodegen all` で明示実行（親はグループに徹する）**: config / profile と一貫し誤実行に強いが、旧 run の打鍵感を損ない冗長なため却下。
