# 0064. サンプルアセット音源をリポジトリに同梱し GitHub Release へ自動添付する（ADR-0041 改訂）

- ステータス: 採用
- 日付: 2026-06-14

## コンテキスト

アセット設定ステップ追加で「すぐ音入りで試したい」ニーズが生まれた。ADR-0041 は音声同梱を容量増・ライセンス管理を理由に却下したが、再検討の結果、増えるのはリポジトリ容量のみで本体バイナリは不変、ライセンスも CC0 音源＋クレジット同梱で対処できると判断した。

## 決定

CC0 等の音源をリポジトリに同梱し、リリース時に GitHub Release へ zip を自動添付する。ADR-0041 の同梱却下を本件に限り改訂する。

- 音源とテキスト（`assets.yaml`・README・`CREDITS`）を `sample-assets/` に置く。
- GoReleaser で zip を生成し Release へ自動添付する。
- ユーザーは `releases/latest/download/` から取得、またはクローンで入手する。
- 音源調達・ライセンス確認はメンテナが行う。

## 結果

- 導入の手間が下がり、すぐ音入りで試せる。
- 本体バイナリは肥大化しない（zip は別アセット、go:embed しない）。
- リポジトリ容量は増え、音源調達・ライセンス確認の人手作業は残る。

## 検討した代替案

- **外部・非同梱配布**: 手動添付が残り自動化の意向に合わず却下。
- **go:embed 同梱**: 本体が肥大化するため却下。
- **ffmpeg 合成**: 実音源を求める目的に合わず却下。
