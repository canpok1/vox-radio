# 0047. 番組タイムゾーン設定の導入と時間文脈（記事配信日時・収録日時）の生成LLMへの伝達

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

記事を紹介する台本で「公開されたばかり」「先日配信された記事」のような時間文脈を踏まえた表現をしたい（Issue #325）。現状 LLM に渡る記事情報は URL/タイトル/要約/要点 のみで、記事の配信日時・台本生成時の収録日時・番組のローカルタイムゾーンはデータ構造に存在せず渡っていない。`ProgramConfig` に timezone 項目は無く、write ステップへ現在時刻を渡す経路も無い。

記事メタ情報は rundown を正として伝播させ（ADR-0015）、rundown 経由で LLM に渡す前例がある（ADR-0037 出演回数、ADR-0046 出典）。本 ADR は出典とは別に、時間文脈と番組 TZ 設定の扱いを決める。

## 決定

- **番組 TZ 設定**: `ProgramConfig` に `Timezone`（IANA tz 名）を追加。デフォルト `Asia/Tokyo`、不正値は WARN ＋ `time.UTC` フォールバックで処理を止めない。
- **配信日時**: `Article`/`RundownArticle` に `Published`（`omitempty`）を追加し、出典と同じ経路で rundown へ流す。取得元は gofeed `item.PublishedParsed`、nil ならベストエフォートで空。
- **時刻粒度・TZ**: 「公開されたばかり」等の話題作りのため日付のみでなく**時刻まで含む RFC3339** で保持し、番組 TZ に変換して収録日時と表示を揃える。
- **収録日時**: 台本生成時の `time.Now()` を番組 TZ で整形し、TZ 名とともに write プロンプトへ渡す（`{{recorded_at}}`/`{{timezone}}`）。
- **台本での扱い**: 使い方は LLM に委ね、RFC3339 文字列はセリフに読み上げず自然な日本語表現に変換する（config-prompt.md と整合、読み変換は既存 direct が担当）。

## 結果

- 記事の新しさ・配信タイミングを踏まえた台本が可能になり、番組のローカル時刻を設定で制御できる。
- `omitempty` で空値は出力されず後方互換を保つ。一方 collect/write へ `*time.Location` を注入する配線が増え、直接 URL 記事は配信日時を取得できず空となる。

## 検討した代替案

- **日付のみ（YYYY-MM-DD）で渡す**: 時刻粒度の話題が作れず不採用。
- **配信日時/収録日時を別 Issue に分離**: 番組 TZ 設定を介して結合し Issue 間依存が生じるため #325 に集約。
- **フィードの元 offset のまま渡す**: 収録日時と表示 TZ が揃わないため、番組 TZ で統一する方針を採用。
