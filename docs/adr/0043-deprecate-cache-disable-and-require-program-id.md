# 0043. キャッシュ無効化機能（cache.enabled）を廃止し program.id を必須化する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

エピソード履歴キャッシュは `cache.enabled`（デフォルト無効）で無効化でき、有効判定は `cfg.Cache.Enabled && p.Program.ID != ""` の二重条件になっている。実体は `program.id` ごとの JSONL のため、`program.id` が空だとフラグが有効でもキャッシュは動かない。

無効化フラグは分岐とテストを増やす一方で意義が薄い。回番号・出演回数・条件付きコーナー/ゲスト（ADR-0031/0032/0037/0040）はキャッシュ有効が前提で、無効時は機能が縮退するだけである。`program.id` 未設定ではキャッシュキーが存在せず保存できない構造的制約もある。関連 Issue: #311。

## 決定

`cache.enabled` を完全削除し、`program.id` を必須化してキャッシュを常時有効にする。`ProgramConfig.ID` の `omitempty` を外し、未設定は `check`・`generate` の両経路でバリデーションエラーとする。有効判定の分岐は撤去して無条件化し、ADR-0031/0032/0037 の「`cache.enabled` が前提」という記述を「`program.id` 必須・常時有効」に更新する。

## 結果

分岐とテストが減り、回番号・出演回数・条件付き機能が常に有効化されて挙動が一意になる。「フラグ有効だが program.id 未設定で動かない」という不整合も解消する。

一方で破壊的変更となる。strict 解析の `check` では既存 `vox-radio.yaml` の `cache.enabled` 行削除が必要で、`program.id` 未設定の episode-spec はエラーになる。移行手順を README/リリースノートに明記する。

## 検討した代替案

- **`enabled` を deprecated として残し常に有効扱い**: 後方互換だがフィールド定義が残り「廃止」が不完全になる。簡素化の目的に反するため却下。
- **`program.id` を任意のまま常時有効化**: キャッシュキーが定まらず保存先を決められない。構造的に成立しないため却下。
