# 0071. slackpost のメッセージ整形を text/template ベースの2テンプレ構成へ刷新する

- ステータス: 採用（テンプレの持ち方は [ADR-0072](0072-render-command-and-template-file-reference.md) で改定: yaml インライン → テンプレファイルのパス参照）
- 日付: 2026-06-15

## コンテキスト

`slackpost` のメッセージ整形は `slack-spec.yaml` の `header` / `fallback` / `summary` / `corner` / `article` の5テンプレを `strings.ReplaceAll` で展開する場当たり実装になっている。条件分岐・ループ・コーナー単位の出し分けができず、`「」`除去・`第0回`除去・Slack リンク剥がし・空行除去などの特殊処理を Go コードへ if 文で積み増す構造になっている。

このため「URL なし記事（お便りコーナーの投稿）を記事リストから除外したい」という要望を設定だけでは満たせない。要望元は紹介記事（URL あり）だけを残したい。

## 決定

メッセージ整形を Go 標準の `text/template` ベースへ全面移行し、テンプレ構成を次の2種類へ再編する。

- **親メッセージ（`parent`）**: mp3 アップロード時の initial_comment。音声ファイル添付あり。
- **スレッド内メッセージ（`thread`）**: スレッド返信本文。音声ファイル添付なし。

詳細方針:

- 両テンプレで使えるプレースホルダ（テンプレのデータ文脈）は共通とし、manifest 全体（`Corners` / `Credits` / `ConversationNotes` / `Casts` 等を含む）を公開する。
- コーナーは `{{range .Corners}}` で全件展開できるほか、テンプレ関数 `corner "<id>"` で manifest の `corner.id` を指定して特定コーナーを任意位置へ埋め込める。
- URL なし記事のスキップ・空コーナーの出し分けは `{{if .URL}}` 等の条件分岐でテンプレ側に表現する（コードへ特殊処理を積まない）。既定テンプレは URL なし記事をスキップする。
- スレッド本文は1テンプレを1つの Markdown 文字列としてレンダリングし、コードが Slack の1 Section ブロック=3000文字制限に合わせて改行境界で自動分割し複数ブロック化する。
- 旧記法（`header`/`summary`/`corner`/`article` 等）は廃止する破壊的変更とし、互換維持・移行ツールは設けない（利用者がごく少数のため）。`init` 再生成で新テンプレを提供する。`init` が生成する `slack-spec.yaml` には、初見ユーザーが設定を書けるよう各テンプレ・プレースホルダ・コーナー出し分け等の使い方を YAML コメントで丁寧に付ける。

## 結果

- 条件分岐・ループ・コーナー単位の出し分けがテンプレ設定だけで完結し、要望の URL なし記事スキップを含め以後の整形要件をコード改修なしで吸収できる。
- `「」`除去・`第0回`除去・リンク剥がし等の Go 側特殊処理を撤去でき、整形ロジックがテンプレへ一元化される。
- `slack-spec.yaml` は破壊的変更となり、既存設定は書き直しが必要。`slackpost check` で text/template の構文検証を行う。

## 検討した代替案

- **現行記法のまま拡張**（コーナーid上書き・URLスキップ設定を追加）: 互換は保てるが ad-hoc な特殊処理が増え続け、根本課題（表現力不足）が残る。
- **URL なし記事の一律スキップをコードに固定**: 今回の要望は満たすが、出し分けの柔軟性がなく次の整形要望でまたコード改修が必要になる。
