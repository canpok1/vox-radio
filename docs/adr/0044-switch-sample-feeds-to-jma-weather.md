# 0044. サンプル設定のデータソースを気象庁防災情報XMLフィードへ変更する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

`init --sample` で生成するサンプル番組（ニュースバラエティ）は Yahoo!ニュース・NHK の RSS をデータソースにしていたが、両者の規約は私的利用に限定されており、RSS を組み込んだプログラムの公開・再配信を許可していない。vox-radio はフィードの記事を LLM で要約・翻案して番組化（公開配信も想定）するため、サンプルのまま使うと規約抵触リスクがある。民間気象サービス（tenki.jp 等）や Zenn・Qiita 等のフィードも、要約・番組化の明示許諾はなくグレーである。一方、気象庁ホームページのコンテンツは公共データ利用規約（第1.0版、CC BY 4.0 互換）に基づき、出典明記のうえで複製・翻案・再配信・商用利用が可能。ただし防災情報XMLの定時フィード（天気概況・府県天気予報）は entry 本文がラベルのみで実文が別 XML にあり、title と description だけを使う collect 実装では利用できない。読み上げ可能な本文を持つのは随時フィード（警報・注意報）と地震火山フィードに限られる。

## 決定

サンプル番組を「ずんだもんとめたんのお天気ラジオ」（program id: `weather-radio`）に衣替えし、データソースを気象庁防災情報XMLフィードへ変更する。メインコーナーは随時フィード（extra.xml、警報・注意報）、日替わりコーナーの1つに地震火山フィード(eqvol.xml)を使い、残る日替わり2つ（お天気豆知識・防災ワンポイント）は source なしで LLM の知識から生成する。出典表記は `feed-spec.yaml` の `feed.credit` に「気象庁「防災情報XML」を加工して作成」を記入して規約の出典明記・加工記載要件を満たす。コーナー数・尺（計300秒）・日替わりローテーション（every:3）・ゲスト出演（every:5）などの機能デモ構成は維持する。

## 結果

サンプル設定をそのまま公開配信に使っても規約抵触の懸念がなくなり、`credit` フィールドの出典表記が CC BY 系データ利用のお手本にもなる。collect の実フィード検証（15記事取得）も成功している。一方、「今日の天気予報を読む」コーナーは現行 collect 実装（フィード本文のみ使用）では作れず、警報・注意報ベースの構成になる。また随時フィードは同一内容の電文が重複するため、重複集約をコーナーの content 指示で LLM に委ねている。

## 検討した代替案

- **現コンセプト維持で規約のゆるい一般ニュースfeedへ差し替え**: 私的利用を超えて要約・再配信できる日本語の一般ニュース・スポーツ・エンタメfeedが実質存在せず断念。
- **テック番組化（Zenn・Qiita 等）**: フィード購読自体は一般的だが、要約・番組化・再配信の明示許諾がなくグレーのため却下。
- **ウィキニュース日本語版（CC BY）**: ライセンスは明確だが更新がほぼ停止しており、サンプルとして毎回同じ記事になるため却下。
