# 0040. 出演回数を「今回を含めた回数（初登場=1）」で永続化し、LLM 入力は境界で逆変換して維持する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

ADR-0037 では `RundownCast.AppearanceCount` を「今回を含まない過去の出演回数（0 = 初登場）」と定義し永続化した。しかしこの値を直接読む人間には「その回を含めて何回目か（初登場 = 1）」の方が直感的で、永続データの可読性に課題があった（Issue #293）。

一方、LLM への渡し方は変えたくない。現状の注記・文言・数値の意味（今回含まず・0 = 初登場）は運用実績があり数字の意図が正しく伝わっているためである。LLM に `RundownCast` を渡す経路は select・flow（生 JSON）と write（テキスト）の 3 つあり、いずれも現行前提で書かれている。

## 決定

- **永続定義を変更**: `AppearanceCount` を「その回を含めた出演回数」とし初登場 = 1。`selectCasts` で過去集計値に 1 を加算して永続化する。
- **LLM 入力は境界で逆変換**: シリアライズ直前に「1 を引いた値（0 クランプ）」へ変換し、従来入力を 3 経路すべてで再現する。プロンプト 3 ファイルは無編集。
- **逆変換を共通ヘルパーに集約**: `RundownCast.PastAppearanceCount()` と一括変換 `CastsForLLM()` に寄せ重複を避ける。
- 本 ADR は ADR-0037 のうち回数の定義のみを置換する（記録方式・rundown 焼き込み・キャッシュ集計は踏襲）。

## 結果

永続データが人間に直感的になり、LLM 入力は現状と完全一致するため生成品質の回帰リスクがない。逆変換が単一ヘルパーに集約され横展開漏れを防げる。トレードオフとして永続値（1 始まり）と LLM 入力値（0 始まり）が二重定義になるが、その「なぜ」を本 ADR で明示して緩和する。

## 検討した代替案

- **LLM 入力も新定義に追従（境界変換なし）**: 二重定義は避けられるが、実績ある LLM 入力が変わり生成品質の回帰リスクがあるため却下。
- **現状維持（ADR-0037 のまま）**: 永続データの可読性課題が残るため却下。
