# 0003. 台本生成を多段 LLM パイプラインに分割する

- ステータス: 採用
- 日付: 2026-05-30

## コンテキスト

台本生成は軽量モデル（Gemini Flash 系）を前提とする。軽量モデルは長文入力や複雑な一括指示で精度が落ちやすい。番組全体を 1 コールで生成すると、入出力が大きくなり破綻しやすく、不調時に全体をやり直す必要がある。

## 決定

台本生成を **1 コール = 1 責務**の多段パイプラインに分割する: `[0] 記事要約(map) → [A] テーマ決定(rundown) → [B] 台本生成(コーナー単位 fan-out) → [C] 演出`。各ステップは専用プロンプト・専用スキーマ・検証・リトライを個別に持つ。`[C]` 演出は**ルール中心でコードが決定的に組み立て、創造的判断（SE 挿入位置）のみ LLM に任せる**（OP/ED ジングル・BGM・話者割当・話速等はコード固定）。コーナー間のトーンは共通のキャラ設定（show.yaml）と rundown を各 `[B]` に渡して揃え、**通し調整パスは設けない**。

## 結果

- 各コールの入出力が小さく保たれ、軽量モデルでも安定する。`[B]` は並列実行できる。
- 不調なステップだけを再実行でき、局所修正（尺超過時の該当コーナー再生成等）が容易。
- `[C]` の LLM 責務を最小化することで演出の安定性が増す。
- ステップ数が増え、中間生成物の管理（→ ADR 0004）が前提になる。

## 検討した代替案

- **単一プロンプトで一括生成**: 実装は単純だが軽量モデルで破綻しやすく、局所再実行もできないため却下。
- **通し調整パスを追加**: トーン統一に有効そうだが、軽量モデルでは逆効果になりうるうえコールが増えるため却下。
