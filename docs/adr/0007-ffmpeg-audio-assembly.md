# 0007. 音声整形に ffmpeg を採用する

- ステータス: 採用
- 日付: 2026-05-30

## コンテキスト

合成済みのセリフ WAV 群（③ synth の出力）を、BGM・効果音・ジングルと重ね合わせて 1 本の番組音声にする必要がある。要件には、しゃべり中だけ BGM を下げるダッキング、スマホ視聴向けのラウドネス正規化、mp3 エンコードが含まれる。これらを自前実装するのは負担が大きい。

## 決定

音声整形は **ffmpeg** に集約する。speech 連結（間に無音挿入）・SE/ジングル配置（`adelay`/`amix`）・BGM ループ（`aloop`）＋音量調整＋ダッキング（`sidechaincompress`）・ラウドネス正規化（`loudnorm`, EBU R128）・mp3 エンコードを、Go 側で組み立てた**単一の `filter_complex` で 1 回の ffmpeg 呼び出し**として実行する。各 clip の尺と最終 mp3 の尺・バイト数は `ffprobe` で計測する。

## 結果

- 放送演出に必要な処理を実績ある 1 ツールで賄え、自前の音声処理を持たずに済む。
- 単一 `filter_complex` にすることで中間ファイルの取り回しが減り、堅牢になる。
- ffmpeg / ffprobe が実行環境（devcontainer・GitHub Actions）に必要となる外部依存が生じる。
- フィルタグラフが複雑になりうるため、Go 側の組み立てとテストを丁寧に行う。

## 検討した代替案

- **音声処理を自前実装**: 依存は減るがダッキング・正規化等の品質確保が難しく、実装コストが過大なため却下。
- **複数ツールの併用**: 段階ごとに最適ツールを選べるが、依存と中間ファイルが増え運用が複雑になるため却下。
