# 0086. 単発番組モード（single_shot）で回番号を無効化し命名・キャッシュの例外を導入する

- ステータス: 採用
- 日付: 2026-06-19

## コンテキスト

vox-radio はキャッシュ履歴から `episode_number` を自動採番し（`cache.NextEpisodeNumber`）、タイトル「第N回」・manifest・ファイル名 `{program.id}_ep{NNN}.mp3`（ADR-0067）に露出させる。

しかし連続性のない単発番組（デモ番組など）では「第1回」と番号が付くと違和感がある。`script_note` で音声・要約への露出は抑えられるが、構造フィールド（manifest・ファイル名・feed の「第N回」）には残り、設定で無効化する手段がなかった。

## 決定

`program` に単発番組モード `single_shot`（デフォルト `false` ＝従来採番・後方互換）を追加し、有効時は以下を一括で適用する。

- 採番をスキップし `episode_number=0`（無効値）をパイプラインへ供給する。タイトルは fallback（番組名）、manifest は `omitempty` で省略、write プロンプトの「今回の放送回」セクションはセクションごと出力しない。
- 出力ファイル名を**サフィックス無し**（`{program.id}.mp3` / `_manifest.json`）にする（ADR-0067 の `_ep{NNN}` 命名の**例外**。判定は `episodeNumber<=0`）。
- キャッシュ（`.vox-radio/cache/{id}.jsonl`）に保存しない。結果として feedgen（キャッシュ由来）の RSS にも載らない。毎回同名で上書き運用となる（再生成は `--force`）。
- 回番号ベースの出現条件（`corners[].condition` / `casts[].condition`）とは併用不可（設定バリデーションでエラー）。

デモ番組（`demo/episode-spec.yaml`）に `single_shot: true` を適用する。

## 結果

- 単発番組は番号・連載の概念から解放され、`{program.id}.mp3` 単体配布に適した成果物になる。
- ADR-0067 の「全経路で `_ep{NNN}` 一貫」に例外が1つ増える。命名分岐は `EpisodeBaseName` の `episodeNumber<=0` 判定に集約し、影響範囲を最小化した。
- キャッシュ未保存のため過去回履歴・RSS 連載に載らない。単発番組は mp3 単体配布前提のためこれを許容する。
- トレードオフ: 通常モードと単発モードで命名規則・キャッシュ挙動が分岐する。フラグ1つに集約して認知負荷を抑えた。

## 検討した代替案

- **`episode_number` を固定値で設定可能にする**: 「第0回」等の不自然な番号が残り、ファイル名・feed の番号も消せないため却下。
- **番号無効・ファイル名・キャッシュ抑制を個別フラグにする**: 単発番組では3つが常に同時に必要で、組み合わせ爆発と設定ミスを招くため、1フラグに集約した。
- **feed 側で番号を無効化する対応も実装する**: キャッシュ未保存で自動的に RSS から除外されるため不要。実装しない。
