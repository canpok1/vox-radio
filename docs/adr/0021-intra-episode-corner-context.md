# 0021. コーナー台本生成に同一回の生成済みセリフを文脈として渡す

- ステータス: 採用
- 日付: 2026-06-01

## コンテキスト

台本生成（`internal/script/write`）はコーナーごとに独立して LLM を呼ぶ。`LLMScriptGenerator.writeAll`（`internal/script/script.go`）と CLI の `runScriptWrite`（`internal/cli/script.go`）はいずれも単純な `for` ループで各コーナーを生成し、結果スライスに貯めるだけで、**直前までに生成した実セリフを次コーナーの `Write` へ渡していない**。

各コーナー生成時に LLM が知れるのは `{{program}}` に含まれる**全コーナーのアウトライン（`title`+`content` のみ）**であり、「オープニングで実際に何を喋ったか」という生成済みの会話本文は不明。このため、オープニングで話した内容を次のコーナーで再度話すなど、**同一回内での内容重複**が発生していた。

ADR-0019（会話メモ）は `{{past_episodes}}` 経由で**別日エピソード間**の連続性を担保する仕組みであり、**同一回の中（オープニング→各コーナー）の文脈分断**は対象外で、本件は未解決だった。

## 決定

- `write.Writer.Write` のインターフェースに、**それまでに生成済みのコーナー（`title`＋セリフ列）**を渡す引数 `previousCorners []model.CornerLines` を追加する。
- `writeAll` / `runScriptWrite` のループで生成済みコーナーを順に蓄積し、各コーナー生成時に**それより前の全コーナーの生成済みセリフ全文**を文脈として渡す（要約・ダイジェスト化はしない）。
- `regenIfNeeded` で worst コーナーを再生成する際も、そのコーナーより前の生成済みセリフを同様に渡す。
- `prompts/write.md` に `{{previous_corners}}` セクションを追加し、「同一回の番組でここまでに話した内容」であること、**既に話した内容を繰り返さず話を前に進める**ことを指示する。
- LLM へ渡すのは `speaker_role`＋`text` のみ（`style`/`intonation` 等の音声プリセットは文脈に不要なため落とす）。

## 結果

- 各コーナーが同一回の既出内容を把握でき、オープニング等で話した内容の重複を防げる。コールバックや話の連続性も付けやすくなる。
- 追加の LLM 呼び出しは発生しない（既存の write コールに文脈を足すだけ）。
- トレードオフ: 後半コーナーほどプロンプトに積む文脈が増え、番組全体ではセリフ量に対しおおよそ O(n²) のトークン増（n=コーナー数）。ラジオ1回はコーナー数・総文字数とも小さく許容範囲と判断。長尺・多コーナー化した場合は将来、要約ダイジェスト方式への切り替えを再検討する。

## 検討した代替案

- **コーナーごとに要約を生成し、それまでの要約だけを渡す**: トークンを節約できるが、コーナー数分の LLM 呼び出しが増え、要約の損失で「同じ言い回しの重複」を防ぎきれないため却下。総文字数が小さい現状では全文方式で十分。
- **直前1コーナーのみ全文を渡す**: トークン最小だが、離れたコーナー間（例: オープニング↔エンディング）の重複を防げず、目的を十分に満たさないため却下。
- **何もしない（アウトライン共有のみ）**: 現状方式。アウトライン（`content`）は設計意図であって実発話ではないため重複を防げず、本件の課題そのもの。
