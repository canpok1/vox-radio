# 0060. 注入検出記事をフィールド破棄でなく丸ごと除外する

- ステータス: 採用
- 日付: 2026-06-12

## コンテキスト

ADR-0059 で記事の原文（`Body`）を write（セリフ生成）LLM まで引き渡すことにした。これにより、ADR-0057 のサニタイズで注入が検出された際の既定動作「当該フィールドを空文字に落として継続」では問題が生じる。お便り（リスナー投稿）の `Body` が空化されると「中身のないお便り」が write に渡り、キャストが読むものがない状態になる。`Body` はお便りの本体であり、フィールド破棄が前提とする「タイトル＋メタデータのみで価値が残る」（ADR-0057 決定2）がお便りでは成り立たない。

## 決定

`on_detect=exclude`（既定）時、注入パターンが**いずれかのフィールドで検出された記事は、当該フィールドの空化ではなく記事を丸ごと除外**する（rundown 以降に一切渡さない）。不可視文字除去・本文長の切り詰めは、残す記事に対して従来どおり適用する。`on_detect=error` 時はパイプライン停止のまま（ADR-0057 を踏襲）。除外は WARN ログに記録する。本決定は **ADR-0057 の決定2（検出時はフィールド単位で破棄）を改定**する。

## 結果

- 注入を含む記事が下流（summarize/write）に一切届かず、原文を write に渡す ADR-0059 の構成でも安全。
- お便りが空本文で読み上げられる事態を防ぎ、お便りコーナーは `script_note` の「お便りが届いていない」フォールバックに自然に繋がる。
- トレードオフ: 本文のみ汚染で他フィールドが有用な記事も失う（ADR-0057 の「タイトルのみ要約」は得られない）。誤検知時は記事1本がまるごと欠落し被害はフィールド欠落より大きいが、ヒューリスティックは保守的で誤検知は限定的。

## 検討した代替案

- **フィールド単位破棄の維持（ADR-0057 現状）**: `Body` 空化で「中身のないお便り」が write に渡るため却下。
- **`Body` 検出時のみ記事除外し他フィールドは空化**: ルールが複雑化し、注入を含む記事を部分的に信頼することになる。単純で保守的な「検出＝丸ごと除外」を選択。
- **`on_detect=error` を既定化**: ニュース記事でもパイプライン停止し自動配信と相性が悪いため却下。
