# 0031. ゲスト出演を回番号条件で制御し、出演確定結果を rundown に永続化する

- ステータス: 採用
- 日付: 2026-06-04

## コンテキスト

「ときどき番組に登場するゲスト」を設定したい。ゲスト回は全コーナーに通しで出演し、いない回は番組を通して一切出ないこと、とりわけ「なし回なのに途中から急に現れる」事態を構造的に防ぐことが要件。

当初は登場確率（ランダム）方式を検討したが、誰がどの回に出るか事前に確定でき計画しやすい点を優先し、回番号で出現を指定する方式に切り替えた。

既存構造では、出演者は `characters` カタログ（`map[キャラID]`）で定義しコーナーの `cast` で割り当て（ADR-0011）、`02_rundown.json` が回ごとの内容の single source of truth（ADR-0015）、回番号はキャッシュから算出される。

## 決定

- ゲスト設定を `episode-spec.yaml` に **map 形式**（キー=キャラID、既存 `characters`/`cast` の慣習に合わせる）で新設。値は `GuestConfig{ role, condition }`。
- 出現条件は `condition` にネストし、現状は `episodes`（明示リスト）と `every`（周期）の**論理和**。将来 `probability` 等を**後方互換で追加**できる拡張点とする。
- 出演判定は回番号からの**決定論的処理**（乱数なし）。回番号が確定するエントリポイント（`episodegen` / `rundown`）で確定し、`model.Rundown.Guests` に**永続化**する。`script` 系は読むだけで再実行しても構成が変わらない。
- 確定ゲストを**全コーナーの `cast` にマージ**（`ApplyGuests`）してから台本生成へ渡す。これで「通し出演」と「なし回は cast に現れない＝LLM が登場させられない」を構造的に保証する。
- `write.md` の `{{guest_info}}` で、最初のコーナーで紹介・中間は同席継続・最後で見送りを指示する。
- 回番号が不明（キャッシュ無効等で 0）の場合は回番号条件ゲストを出さず警告する。

## 結果

**良い影響**: ゲスト回を事前確定でき計画しやすい。cast マージで通し出演／途中登場防止を構造的に担保。永続化で `script` 単独再実行でも再現する。map と condition ネストで既存慣習との一貫性と将来拡張性を両立する。

**トレードオフ**: 回番号条件はキャッシュ有効（`cache.enabled` かつ `program.id`）が前提。単独 `rundown` コマンドに回番号算出を追加する必要がある。確率によるランダム出演は本決定では提供しないが `condition` 拡張で将来対応できる。

## 検討した代替案

- **登場確率方式**: 意外性は出せるが事前確定できず計画しづらい。確率は将来の `condition` 拡張に委ねて却下。
- **プロンプト指示のみ（cast 非マージ）**: LLM が「ゲストなし」を破る余地が残り最重要要件を保証できないため却下。
- **専用中間ファイルへ永続化**: ゲスト出演者も「この回の内容」の一部であり SoT の rundown（ADR-0015）へ統合。
- **ゲストをスライスで保持**: 順序不要かつキャラ ID が一意のため map が自然。重複も防げるため map を採用。
