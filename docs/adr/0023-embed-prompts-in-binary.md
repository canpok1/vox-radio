# 0023. プロンプトをバイナリに同梱し --prompts フラグを廃止する

- ステータス: 採用
- 日付: 2026-06-01

## コンテキスト

プロンプトテンプレート（`prompts/*.md`、6ファイル）は `loadPrompts(dir)` がカレントディレクトリ基準でディスクから読み込み、`--prompts` フラグ（デフォルト `prompts`）で配置を指定していた。

しかし GitHub Release（GoReleaser）はバイナリのみを配布し `prompts/` を同梱しないため、ダウンロードしたバイナリ単体では `run` 等が `prompts/` 不在で失敗する。プロンプトは番組生成ロジックの一部でありユーザーが編集する必要がないにもかかわらず、実行の前提として外部ファイルの用意を強いていた。

設定（Config/Profile）・アセットはジャンルごとにユーザーが用意する外部ファイル（ADR-0010/0011）だが、プロンプトはこれらと性質が異なる。

## 決定

プロンプトを `//go:embed` でバイナリに同梱し、`--prompts` フラグを廃止する。

- ファイルを `internal/cli/prompts/` へ移動し、`internal/cli/init.go` の `templates/` 同梱と同じパターンで埋め込む（`go:embed` は親ディレクトリを参照できないため移動が必要）。
- `loadPrompts()` は引数なしで `embed.FS` から読み込む形に変更する。
- 設定・アセットは従来どおり外部ファイルのまま維持する。

## 結果

- バイナリ単体で実行可能になり、配布・利用の前提条件が消える。
- プロンプトとコードのバージョン整合がビルド時に保証される。
- 破壊的変更: `--prompts` による実行時の差し替えができなくなる。開発時にプロンプトを変更する場合はリビルドが必要。
- プロンプトはユーザー編集を想定しないため、これらのトレードオフは許容範囲と判断した。

## 検討した代替案

- **`--prompts` を残し埋め込みをデフォルトにする**: 実行時オーバーライドを維持できるが、編集不要なリソースに設定面を残すと CLI とドキュメントが複雑化する。利点が小さいため却下。
- **GoReleaser のアーカイブに `prompts/` を同梱する**: バイナリ単体実行の要望を満たさず、配置パスの前提も残るため却下。
