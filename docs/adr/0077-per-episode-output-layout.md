# 0077. episodegen の出力を回数別名・回ごとディレクトリにする

- ステータス: 採用
- 日付: 2026-06-16

## コンテキスト

ADR-0067 で生成 MP3 を `{program.id}_ep{NNN}.mp3` の回数別名にしたが、マニフェスト（`output/manifest.json`）と中間生成物は固定名のままだった。

ADR-0004 の上書きガードは `episode.mp3` の存在のみを見ていたため、mp3 が回数別名になった後は `episodegen` を複数回実行しても mp3 は衝突せずガードを通過し、固定名のマニフェスト・中間生成物だけが無言で上書きされていた。過去回の成果物を残せず再参照・再配信できない。

## 決定

出力をベース名 `{program.id}_ep{NNN}` で名前空間化する。

- マニフェスト: `<out-dir>/{program.id}_ep{NNN}_manifest.json`（mp3 と名前順で隣接）。
- 中間生成物: `<out-dir>/intermediate/{program.id}_ep{NNN}/` 配下（クリップは同 `05_clips/`）。
- 上書きガード: mp3・マニフェスト・中間ディレクトリのいずれか存在で `--force` なしならエラー終了。`--force` 時は中間ディレクトリを削除して作り直す。
- パス解決は `fileio.EpisodeLayout` のメソッドに集約する。個別サブコマンド（明示パス指定）の挙動は不変。

## 結果

- 複数回実行しても過去回の成果物が保持され、誤上書きを防げる。
- 1 回分が `ep{NNN}` 名・ディレクトリにまとまり管理・掃除がしやすい。
- マニフェスト名が固定でなくなるため、`slackpost --manifest` 等に渡すパスは回数別名で指定する（feedgen は cache 参照のため影響なし）。

## 検討した代替案

- **ガードを manifest・中間にも広げるだけ（命名は固定のまま）**: 最小変更だが回ごとの成果物を残せず、再実行のたびに `--force` か手動退避が要る。却下。
- **mp3 ごと回ごとディレクトリ（`output/ep{NNN}/`）にまとめる**: 1 回分が完全に分離できるが、mp3 のフラット配置（ADR-0067）と配信経路を変える影響が大きい。却下。
