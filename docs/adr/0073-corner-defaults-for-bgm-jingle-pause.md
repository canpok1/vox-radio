# 0073. コーナーの BGM・ジングル・pause に番組共通デフォルト（corner_defaults）を導入する

- ステータス: 採用
- 日付: 2026-06-16

## コンテキスト

ADR 0020 で BGM・ジングルは `CornerConfig` にコーナー単位で決定的に指定する方式へ統一した。再現性は確保されたが、全コーナーで同じ BGM・ジングル・pause を使う場合に毎コーナー同一値を繰り返す必要があり冗長で、一括変更も漏れやすい。

要望は「全コーナー共通のデフォルトを定義し、特定コーナーだけ上書き、または無効化（例: このコーナーは BGM なし）できること」。ADR 0020 は番組全体設定（当時は LLM 駆動）を廃止したが、本件は決定的なデフォルト+上書きであり 0020 の「決定性」方針と整合する。一方で「番組レベル設定の再導入」に当たるため記録する。

## 決定

**`episode-spec.yaml` トップレベルに `corner_defaults:` を追加し、各コーナーが省略時に継承する決定的デフォルトとする。**

- 対象は `bgm` / `start_audio` / `end_audio` / `start_pause_sec` / `end_pause_sec`。
- コーナー値はポインタ型で omit と明示指定を区別し3状態を取る: 省略=継承、値=上書き、空文字（`bgm: ""`、audio は空マッピング `{}`）=無効化。pause は 0 が通常値のため専用の無効化表現は持たない。
- 解決（`resolveCornerDefaults`）は読込・マージ後／検証前に行い、下流は従来どおり concrete 値を受け取る。
- `corner_defaults` 自体に無効化（空指定）は書けない（実在キー or 省略のみ。空指定はエラー）。

## 結果

共通の音楽演出を1箇所で定義でき、繰り返し記述と一括変更漏れが解消する。`corner_defaults` を書かない既存 spec は従来どおり動作し後方互換を保つ。3状態表現のため `CornerConfig` の `BGM`・`StartPauseSec`・`EndPauseSec` がポインタ化し、下流の deref 追従が必要になる（影響は script.go・credits.go・検証に限定）。番組レベル設定は増えるが、継承+上書きの単方向であり ADR 0020 が嫌った二重管理ではない。

## 検討した代替案

**予約キーワード（例: `none`）で無効化**: `bgm`/jingle/se の id 名前空間と衝突しうる（"none" という id を作れてしまう）。空文字は有効な id になり得ず衝突しないため不採用。

**従来どおりコーナー単位で全記述**: 冗長性と一括変更漏れが残り要望を満たせないため却下。

**program レベルに独立した OP/ED 設定を併存**: ADR 0020 が二重管理として却下済み。corner_defaults は単方向継承で二重管理にならないため採用。
