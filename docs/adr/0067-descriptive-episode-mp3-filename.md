# 0067. 生成 MP3 のファイル名を `{program.id}_ep{NNN}.mp3` にする（ADR-0026 改訂）

- ステータス: 採用
- 日付: 2026-06-14

## コンテキスト

`episodegen` が生成する MP3 のファイル名は `internal/fileio/paths.go` で `episode.mp3` に固定されている。ADR-0026 は「`episode.mp3` をリネーム不要で公開し、回の区別はディレクトリ（`ep-N/`）で行う」と決めた。

しかしローカルでは複数の番組・回の成果物がすべて `episode.mp3` となり、ファイル名だけでは番組種別も回数も判別できず管理しづらい。`manifest.audio_file` は実ファイル名と一致している必要があり、`audio_file` は cache → RSS フィード URL（`{audio_file}` テンプレート置換）→ Slack アップロード名へ伝播する。したがってファイル名はこの経路全体で整合させる必要がある。

## 決定

- `episodegen`（オーケストレータ）の最終 MP3 を **`{program.id}_ep{NNN}.mp3`**（回番号は3桁ゼロ埋め、cache 由来の `episodeNumber`）で出力する。
- `manifest.audio_file` をこの名前にし、cache・RSS フィード URL・Slack アップロード名まで**全経路で一貫**させる。`program.id` は既に cache ファイル名（`{id}.jsonl`）に直接使われており、ファイルシステム安全な値である前提を踏襲する。
- 単体サブコマンド `episodegen assemble` / `manifest` は従来どおり `--out` でユーザー指定とし、自動命名しない。
- 上書きガードは `episodeNumber` が必要になるため、cache 読み込みをガードより前へ並べ替える。
- 過去エピソードは cache に旧名（`episode.mp3`）が保存済みのため**移行不要**で、フィードは回ごとの保存名でレンダリングされ影響を受けない。

ADR-0026 の「リネーム不要で公開」は、**`episodegen` が最初から説明的な名前で出力する**形に改訂する（手動リネームが不要な点は維持される）。

## 結果

- ローカルの MP3 がファイル名だけで番組・回を自己説明する。
- 新規回の RSS URL・Slack 名は新形式になるが、回ごとに保存名を持つため既存回は不変。
- 3桁ゼロ埋めは既存 `clip_%03d.wav` 規約と辞書順ソートに合わせる（1000 回超でも桁が伸びるだけで破綻しない）。
- トレードオフ: 単体テスト・e2e feature・fixture・golden・docs に広く波及する。

## 検討した代替案

- **ディスク上の名前だけ変え `audio_file` は `episode.mp3` のまま**: manifest と実ファイルが不一致になり脆い。却下。
- **ゼロ埋めなし（`ep12`）**: 辞書順ソートが崩れるため却下。
- **サブタイトル（`EpisodeTitle`）をファイル名に含める**: 不安定で長く、ファイルシステム非安全文字を含みうるため却下。
