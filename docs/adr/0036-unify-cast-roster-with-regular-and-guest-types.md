# 0036. 出演者ロスターを casts に一本化し、レギュラーと出演条件（お休み）を導入する

- ステータス: 採用
- 日付: 2026-06-05

## コンテキスト

出演者は2か所に分散していた。レギュラーは `corners[].cast`（コーナー固定キャスト）に暗黙的に存在し、program レベルで「この番組のレギュラーは誰か」を宣言できない。条件付き出演はゲスト（ADR-0031）にしかなく、レギュラーは無条件・毎回で「特定回だけお休み」を表現できなかった。レギュラーとゲストは同型の構造（role + 出演条件）なのに、ゲスト側だけ実装されていた。

## 決定

出演者ロスターを `EpisodeSpec.Casts`（`map[キャラID]CastConfig`）に一本化し、旧 `Guests` を廃止する。`CastConfig` は `type`（`regular`/`guest`）・`role`・`condition`（`*EpisodeCondition`）を持ち、レギュラーとゲストを同一構造で表現して種別フィールドで区別する。

- お休みは専用フィールドを設けず、共通 `condition` と既存の `not`（補集合, ADR-0033）で表現する（例 `condition:{not:{episodes:[5]}}` = 5回目以外出演）。`regular` は `condition` 省略時=毎回出演、`guest` は `condition` 必須。
- `corners[].cast` は演出用途として存続。ロスター（番組全体の役割）は全コーナーに出演させ、`corners[].cast`（コーナー固有の役割）と**両方を保持**してプロンプトへ渡す（上書きしない）。LLM が両役割を踏まえた台本を書けるようにする。
- 確定結果は `Rundown.Casts`（`RundownCast` に `type` を追加）へ永続化。選定は回番号からの決定論的処理（キャラID昇順・乱数なし）。回番号不明時は `condition` 付き casts のみ抑止・警告する。
- 後方互換は取らず `guests` を完全廃止して `casts` へ移行する。

## 結果

レギュラー／ゲストを program レベルで一元宣言でき、両者を同じ機構（選定・`Rundown` 永続化・コーナーマージ・`EpisodeCondition`）で扱えて重複が消える。お休みも `not` の再利用で新フィールドなしに表現できる。一方、設定・テスト・examples の移行が必要で、旧 `guests` の設定は読めなくなる。`type` 必須化でバリデーションと演出分岐（ゲスト=紹介・見送り、レギュラー=常連扱い）が増える。

## 検討した代替案

- **guests を残し regulars を別マップで追加**: 同型構造が二重化し選定・マージ・永続化が分岐する。一本化で却下。
- **お休み専用フィールド（rest/break）を新設**: 既存 `EpisodeCondition` の `not` で表現できるため不要。共通 condition + not を採用。
- **corners[].cast を廃止し全出演者を program へ集約**: コーナーごとに変わる役割記述（例: ボケ／解説）が表現しにくくなる。演出用途として存続させ却下。
- **コーナーロールで番組ロールを上書きし単一 role 化**: 番組全体での役割が失われ台本がコーナー文脈に閉じる。両ロール併記を採用し却下。
