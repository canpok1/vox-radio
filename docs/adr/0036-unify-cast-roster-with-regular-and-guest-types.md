# 0036. 出演者ロスターを casts に一本化し、レギュラーと出演条件（お休み）を導入する

- ステータス: 採用
- 日付: 2026-06-05

## コンテキスト

出演者は2か所に分散していた。レギュラーは `corners[].cast`（コーナー固定キャスト）に暗黙的に存在し program レベルで宣言できず、無条件・毎回で「特定回だけお休み」を表現できなかった。条件付き出演はゲスト（ADR-0031）にしかなく、両者は同型の構造（role + 出演条件）なのにゲスト側だけ実装されていた。

## 決定

出演者ロスターを `EpisodeSpec.Casts`（`map[キャラID]CastConfig`）に一本化し、旧 `Guests` を廃止する。`CastConfig` は `type`（`regular`/`guest`）・`role`・`condition`（`*EpisodeCondition`）を持ち、レギュラーとゲストを同一構造で表現して種別フィールドで区別する。

- お休みは専用フィールドを設けず、共通 `condition` と既存の `not`（補集合, ADR-0033）で表現する（例 `condition:{not:{episodes:[5]}}` = 5回目以外出演）。`regular` は `condition` 省略時=毎回出演、`guest` は `condition` 必須。
- 出演集合はその回アクティブなロスターのみとする。`corners[].cast` は演出用途として存続するが、アクティブメンバーへコーナー固有の役割を与える注釈に限定し、出演者を追加しない（休演キャストの情報はプロンプトに渡さない）。各出演者は番組ロールとコーナーロールの両方を保持して渡し、上書きしない。
- 確定結果は `Rundown.Casts`（`RundownCast` に `type` を追加）へ永続化。選定は回番号からの決定論的処理（キャラID昇順・乱数なし）。回番号不明時は `condition` 付き casts のみ抑止・警告する。
- 後方互換は取らず `guests` を完全廃止して `casts` へ移行する。

## 結果

レギュラー／ゲストを program レベルで一元宣言でき、同じ機構で扱えて重複が消える。お休みも `not` の再利用で表現でき、休演キャストはプロンプトに現れない。一方、設定・テスト・examples の移行が必要で旧 `guests` の設定は読めなくなり、`type` 必須化でバリデーションと演出分岐が増える。

## 検討した代替案

- **guests を残し regulars を別マップで追加**: 同型構造が二重化し選定・マージ・永続化が分岐する。一本化で却下。
- **お休み専用フィールド（rest/break）を新設**: 既存 `EpisodeCondition` の `not` で表現できるため不要。共通 condition + not を採用。
- **corners[].cast を廃止し全出演者を program へ集約**: コーナーごとに変わる役割記述が表現しにくくなる。注釈として存続させ却下。
- **コーナーロールで番組ロールを上書きし単一 role 化**: 番組全体での役割が失われ台本がコーナー文脈に閉じる。両ロール併記で却下。
- **corners[].cast を出演集合に合算**: 休演中のロスターメンバーを corner.cast が復活させ情報が漏れる。注釈限定として却下。
