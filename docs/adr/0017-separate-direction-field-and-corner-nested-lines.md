# 0017. 演出説明フィールドを台本生成から分離し、セリフをコーナー単位のネスト構造で保持する

- ステータス: 採用
- 日付: 2026-05-31

## コンテキスト

コーナー定義の `content` には内容説明と演出指示（「ジングルを流す」など）が混在している。`content` は台本生成(write)で LLM にそのまま渡るため、キャラクターが演出内容をセリフとして口走る問題が生じていた。

一方、演出生成(direct)の入力は番組全体のフラットなセリフ列と asset カタログのみで、コーナーごとの演出意図を受け取っていなかった。伝えたい先(direct)に情報が届かず、伝えたくない先(write)に演出情報が混ざる逆転が起きていた。

direct には2経路ある。単体サブコマンド(`script direct`)は `03_lines.json` だけを読みプロファイルを参照しないため、演出説明の供給には中間ファイルへの永続化が前提となる。

## 決定

コーナーに演出専用フィールド `direction` を追加する。`direction` は write には渡さず、direct にだけ渡すことで `content`(台本用)と演出説明の責務を分離する。

演出説明とセリフの対応づけは、壊れやすい番組全体の通し index ではなく、**コーナーごとにセリフを配列として保持するネスト構造**で行う。`03_lines.json` を `{ corners: [{ title, direction, lines }] }` に変更し、direct の挿入位置はコーナー内ローカル index(`corner_index` + コーナー内 `after_line_index`)で参照する。`direction` がファイルに含まれるため両経路で direct に届く。direct の LLM 呼び出しは1回を維持する(BGM 排他・SE 本数の全体制約のため)。

## 結果

- 演出指示がセリフへ漏れなくなり、口走り問題が解消する。
- セリフ増減が他コーナーの index をずらさず、演出挿入位置が安定する。
- `direction` がコーナー単位で1回だけ保持され、`script direct` 単体経路もファイルのみで完結する。
- 破壊的変更を伴う: `03_lines.json` スキーマ、`Director.Direct` シグネチャ、`insertionsSchema`(`corner_index` 追加)、`buildScript`、`direct.md` プロンプト。ADR 0004 の中間生成物規約に従いテスト・ドキュメントを更新する。

## 検討した代替案

- **通し index に演出説明をレンジで対応づけ(`line_index_range`)**: フラット index を維持し変更は小さいが、レンジ境界の同期が壊れやすいため却下。
- **各 `model.Line` に corner フィールドを付与**: セリフはフラットのまま判別できるが、`direction` はセリフ単位でないため別途コーナーメタ情報も必要となりハイブリッドで煩雑なため却下。
- **演出説明を渡さず content の書き方ルールで回避**: プロンプト調整のみで構造変更不要だが、混在の根本原因が残り再発するため却下。
- **direct をコーナーごとに分割呼び出し**: ローカル index と相性は良いが、BGM 排他や SE 本数の全体最適が失われるため却下。
