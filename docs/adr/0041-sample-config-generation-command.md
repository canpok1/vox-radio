# 0041. 番組生成お試し用のサンプル設定一式を init --sample で生成し、リポジトリ同梱の examples/ サンプルを廃止する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

バイナリ配布（GoReleaser）でダウンロードしたユーザーが、番組生成をすぐ試せるよう「記入済みのサンプル設定一式」を求める要望がある。

現状の課題は以下のとおり。

- サンプルはリポジトリ同梱の `examples/tech.yaml` ＋ `examples/assets/`（音声 mp3 を 3 本: jingle / se / bgm）として git 管理されている。GoReleaser のバイナリ単体にはこれらが含まれないため、リポジトリをクローンしないと参照できない。
- 音声バイナリがリポジトリと配布物の容量を増やす。
- `vox-radio init`（フラグなし）は既に `templates/*.yaml` をスケルトン雛形として `//go:embed` で同梱・生成しているが、`"番組タイトル"` 等のプレースホルダで、そのままでは動かせない。

設定・アセットはユーザーが用意する外部ファイルだが（ADR-0029）、編集不要なリソースをバイナリへ同梱してバイナリ単体実行を可能にする方針は既に確立している（ADR-0023 プロンプト同梱、ADR-0039 スキル同梱）。

## 決定

記入済みの「すぐ動くサンプル設定一式」をバイナリに `//go:embed` で同梱し、`vox-radio init --sample` で生成する。

- スケルトン雛形を生成する従来の `vox-radio init`（フラグなし）はそのまま残す。`--sample` はゼロから作るユーザーではなく「まず動かして体験したい」ユーザー向けに、ずんだもん・めたんが MC を務める一般向けニュース番組（バラエティ風）の完成済み構成を生成する。
- サンプルには音声バイナリを含めない（容量削減）。音声バイナリに依存する設定（`corners[].start_audio` / `end_audio` / `bgm`、`assets/assets.yaml` の中身）はコメントアウトした記入例として同梱し、ユーザーが音声素材を用意してから有効化できるようにする。一方 `episode-spec.yaml` の `assets_files` 参照は有効のままとし、中身が空（全コメント）の `assets/assets.yaml` を指す。これにより spec → アセット設定ファイルの配線をサンプルで示しつつ、ユーザーは `episode-spec.yaml` を編集せず `assets.yaml` のコメントを外すだけで音を追加できる（音声バイナリ非同梱でも `check` は空アセットとして通る）。
- リポジトリ同梱の `examples/` ディレクトリ（`tech.yaml`・音声 mp3・README）を廃止する。`Makefile` の `run-sample` / `check-samples`、README、各コマンドの `Long` 例の参照は、`init --sample` で生成したサンプルを用いる形に置き換える。

## 結果

- バイナリ単体でサンプルを取得でき、リポジトリのクローンも音声同梱も不要になる（ADR-0023 のプロンプト同梱と同じ方針をサンプルへ適用）。
- 音声を含まないため、生成直後は境界音声・BGM 抜きの構成で動作する。音声を使う場合はコメントを解除し素材を別途用意する。
- `examples/` 廃止に伴い、`Makefile`・README・コマンド `Long` 例・`docs/cli/` の参照更新が必要になる（実装 Issue で対応）。
- サンプルとコードのバージョン整合がビルド時に保証され、CI（`make check-samples`）で生成サンプルを検証することでドリフトを検知できる。

## 検討した代替案

- **`examples/` を新ニュースサンプルへ置き換えて維持する**: バイナリ単体では依然参照できず、音声同梱による容量問題も残るため却下。
- **`init`（フラグなし）の雛形自体を記入済みサンプルにする**: 自分の番組をゼロから組み立てたいユーザー向けの白紙スケルトンが失われる。スケルトンとサンプルは目的が異なるため、フラグで分離した。
- **サンプルにダミー音声を同梱する**: 容量増とライセンス管理の手間が生じる。アセット設定のコメント化で代替できるため却下。
