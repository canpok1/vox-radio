# 0070. mp3 エンコード代替を見送り ffmpeg 依存を維持する

- ステータス: 採用
- 日付: 2026-06-15

## コンテキスト

音声整形は ffmpeg に集約している（[ADR-0007](0007-ffmpeg-audio-assembly.md)）。Apple Silicon での ffmpeg 入手が手間という課題から、依存を外せないか調査した。主な事実は次のとおり。

- ffmpeg は mp3 化だけでなく loudnorm・ダッキング等の DSP も担い、mp3 化を外しても DSP が残り自前実装が要る。
- LAME も arm64 の信頼できる公式バイナリ配布がなく、ffmpeg と同じ入手しづらさが残る。
- 純 Go の mp3 エンコーダは実質 shine-mp3 のみだが、採用実績が乏しく品質も LAME 劣り前提。

## 決定

ffmpeg 依存を維持し、mp3 エンコード代替（LAME 置換、shine-mp3 純 Go 化、外部ツール同梱/自動取得）への移行は見送り、ADR-0007 を継続する。

## 結果

- 実績ある単一ツールで放送演出を賄え、DSP の自前再実装を抱えずに済む。
- Apple Silicon での導入の手間は未解決で残る。将来必要なら ffmpeg バンドル/自動取得を別途検討する。

## 検討した代替案

- **LAME 外部 CLI に置換**: arm64 の入手しづらさが ffmpeg と同様で、DSP の自前実装も生じ利点が乏しい。
- **LAME の同梱/自動取得**: 導入は楽だが未対応環境/オフラインの穴と LGPL 表示義務対応が要る。
- **shine-mp3 で純 Go 化**: 外部依存ゼロだが採用実績が乏しく品質も劣る。
