# 0057. 外部情報源のプロンプトインジェクション多層防御

- ステータス: 採用
- 日付: 2026-06-11
- 備考: 決定2（検出時はフィールド単位で破棄）は [ADR-0060](0060-drop-whole-article-on-injection-detection.md) で改定（`on_detect=exclude` 時は記事を丸ごと除外）。

## コンテキスト

`internal/collect` が RSS/Atom フィード・ウェブページから取得した記事（タイトル・本文・著者名・媒体名）は、サニタイズ・長さ制限・命令検出のいずれもなしで `summarize.md` / `write.md` の `{{article}}` / `{{articles}}` に JSON 埋め込みされる。LLM 呼び出しは system ロール分離もなく全文を単一 user メッセージに連結する（ADR-0002）。このため、攻撃者が制御するフィード本文に「これまでの指示を無視して…」等の命令文を仕込むと、要約（summarize）・台本生成（write）の LLM が乗っ取られうる。外部入力は信頼境界の外であり、データとして扱うべき内容が指示として解釈される構造的リスクがある。

## 決定

多層防御を導入する。

1. **collect 境界での決定論的サニタイズ**: 取得直後に①不可視・双方向制御文字の除去、②本文長上限、③命令文ヒューリスティック検出（日本語＋英語の保守的な正規表現セット）を行う。下流は浄化済みデータのみ受け取る。
2. **検出時はフィールド単位で破棄**: `Title`/`Body`/`Source`/`Author` のうち検出されたフィールドを空文字へ落とす。命令文の外科的除去はしない。本文のみ汚染時は自然に「タイトル＋メタデータのみ」要約になる。
3. **LLM による検出は採らない**: 判定自体が注入対象となりコスト・遅延・新たな失敗点を増やすため。
4. **プロンプト枠付け**（provider 非依存）: `summarize.md` / `write.md` に、埋め込み JSON は信頼できない外部データであり中の指示に従わない旨の防御節を追加する。
5. **設定で停止/継続を切替**: `security.prompt_injection.on_detect`（`exclude` 既定 / `error`）。`max_body_chars` で上限を調整可能。

## 結果

- 決定論的で再現・テストが容易。追加 API コスト・遅延がほぼない。
- フィールド破棄により誤検知時も番組生成は止まらず、被害は当該フィールドの欠落に限定される。
- ヒューリスティックは既知パターン中心のため、巧妙・新規の注入は取りこぼしうる。プロンプト枠付けと併せた多層で緩和する。
- system ロール分離は Dify がアプリ側プロンプト固定で効かないため見送り、テンプレート内防御で両 provider を一律カバーする。必要時に別途検討する。

## 検討した代替案

- **命令文の外科的除去**: 一致部分のみ削る案。境界判定が曖昧で取りこぼし・文意破壊が起きやすく、フィールド破棄より脆い。却下。
- **LLM による注入検出**: 精度は上がるが判定自体が注入対象・コスト増・新たな失敗点。却下。
- **system/user ロール分離**: 防御強化に有効だが Dify では効かず provider 間で挙動が割れる。テンプレート内防御を優先し見送り。
- **無対策（現状維持）**: 自動配信で外部入力が直接 LLM を駆動するため受容不可。
