# 0010. 設定を共通設定(Config)とジャンル別設定(Profile)に二分割する

- ステータス: 採用
- 日付: 2026-05-30

## コンテキスト

従来は `config/` 配下に5つのYAML（`feeds.yaml` / `show.yaml` / `assets.yaml` / `llm.yaml` / `podcast.yaml`）が分散し、音声素材は `assets/` で別管理されていた。`assets.yaml` 内のファイルパスは実行時カレントディレクトリ基準で解決されるため、任意のディレクトリからの実行に対応できなかった。ラジオの「ジャンル」（技術ニュース、動作確認用など）を切り替える際に複数ファイルとカレントディレクトリを意識する必要があり、操作が煩雑だった。

## 決定

設定を2種類に分割する。

- **共通設定 (Config)**: LLMのみ。リポジトリ直下の `vox-radio.yaml` に配置し、`LoadConfig(path)` で読み込む。
- **ジャンル別設定 (Profile)**: feeds / show / assets / podcastを1ファイルに統合。`profiles/<genre>/profile.yaml` に配置し、`LoadProfile(path)` で読み込む。

`LoadProfile` はassetの `file:` フィールドがプロファイルファイルのあるディレクトリ基準で相対解決される（絶対パスはそのまま）。CLI各コマンドには `--profile` フラグ（既定: `profiles/test/profile.yaml`）を追加し、`script` コマンドには `--config` フラグ（既定: `vox-radio.yaml`）も加える。旧 `config/` ディレクトリと `assets/` ディレクトリは廃止。

## 結果

- ジャンルの切り替えが `--profile` フラグの変更のみで完結し、カレントディレクトリ非依存になる。
- 共通設定とジャンル別設定の責務が明確に分離される。
- `Config` 型が縮小し（LLMのみ）、`Profile` 型が新設される破壊的変更となる。
- 旧 `--config <dir>` の意味がファイルパスへ変わり、既存の手順書・スクリプトの更新が必要。

## 検討した代替案

- **ディレクトリ指定方式を維持しつつ統合**: `config/` 配下を1ファイルに統合する案。ディレクトリ指定のままではカレントディレクトリ問題が残るため却下。
- **環境変数でジャンルを切り替える**: `PROFILE=tech` などの環境変数を導入する案。ファイルパス指定と比較して可視性が低く、複数ジャンルの並行実行が難しいため却下。
