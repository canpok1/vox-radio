# 0029. アセット設定のプロファイルからの分離

- ステータス: 採用
- 日付: 2026-06-03

## コンテキスト

現在、番組アセット（jingle / SE / BGM）の設定は `profiles/<genre>/profile.yaml` 内の `assets:` セクションにインライン定義されている（ADR-0010 でプロファイルに統合した）。

同じアセット集合を複数番組で共有したいケースが多く、番組ごとに同一内容の `assets:` を重複記載するのは非効率である。アセットだけを配布・共有したいというニーズもある。

ADR-0010 はアセットをプロファイルに統合し、`file:` をプロファイルディレクトリ基準で解決すると決定していた。本件はその一部（アセットのインライン定義）を変更する。

## 決定

アセット設定をプロファイルから分離し、独立したアセット設定ファイル（`assets_files` リスト）として参照する形式に変更する。

- `Profile` 構造体の `Assets AssetsConfig` を `yaml:"-"`（デコード非対象）に変更し、`AssetsFiles []string yaml:"assets_files"` を追加する。
- `assets_files` の各パスはプロファイルファイルのディレクトリ基準で解決する。
- アセット設定ファイル内の `file:` 相対パスは**そのアセット設定ファイルのディレクトリ基準**で解決する（素材とアセット設定ファイルをひとまとめに配布可能）。
- 複数ファイルを後勝ちでマージする（共通アセット集＋番組固有の組み合わせが可能）。
- `LoadProfileStrict` では旧 `assets:` が unknown field エラーになり、移行を強制できる。

## 結果

**良い影響**: アセット集を1ファイルにまとめて複数プロファイルから参照・配布できる。利用側（`p.Assets` 経由）は無変更で済む。

**悪い影響**: 破壊的変更。既存プロファイルの `assets:` を `assets_files:` + 分離ファイルへ移行が必要。`assets_files` 未指定時はアセット空となるため、アセット必須のコマンドは既存の `ValidateProfileAssets` で検出される。

## 検討した代替案

**インライン `assets:` を残しつつ `assets_files` を追加（後方互換）**: 両方サポートするとマージ順序が複雑になり、設定の二重管理を招く。完全分離（破壊的変更）の方が長期的に単純であるため却下。

**プロファイル継承（`extends: base.yaml`）**: より汎用的だが設計コストが高く、本件の目的（アセット共有）に対してオーバースペック。アセット特化の `assets_files` で十分なため却下。
