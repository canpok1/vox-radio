# 0093. 固有名詞の読み方辞書（静的辞書）を LLM 読み変換の前段に追加する

- ステータス: 採用
- 日付: 2026-06-30

## コンテキスト

VOICEVOX（OpenJTalk）の誤読対策として、ADR-0027 で direct にかな化を集約し、ADR-0045 で発音校正パスを追加してきた。いずれも LLM の文脈判断に依存する仕組みであり、複雑な文脈には強い一方、毎回 LLM が同じ固有名詞を正しく変換する保証がない（確実性の課題）。

ADR-0045 では代替案として「VOICEVOX ユーザー辞書の成長方式（静的辞書）」を、(1) かな化と役割が重複する、(2) 静的辞書は文脈依存の読みに対応できない、という理由で不採用とした。

しかし運用上、人名・作品名・略語など「常に同じ読みでよい既知の固有名詞」が一定数存在し、これらは文脈判断が不要なまま LLM 任せになっていた。番組ごとに繰り返し登場する固有名詞を確実に正しく読ませたいという要求があり、ADR-0045 の静的辞書不採用の判断を、適用範囲を限定して見直す必要が生じた。

## 決定

`vox-radio.yaml` にトップレベルの読み方辞書 `pronunciation`（`表記: 読み方` のマップ）を追加し、direct の最終テキスト確定（`buildScript()`）で、元テキストに対して LLM 読み変換マップ（`line_conversions`）を適用する**前段**に辞書置換を挟む。処理フローは「元テキスト → 辞書置換 → LLM 変換マップ適用」とし、LLM が変換した行は LLM 結果を優先する（辞書は LLM が素通しした行に効く）。

辞書は全エピソード共通のグローバル設定とし、同一語彙は常に同じ読みに統一する（複数読み・文脈判断には対応しない）。表記の重複定義（読みの衝突）は YAML の重複キーとして読み込み時にエラーで検出する。辞書置換は決定的にプログラムが行い、Go の map はイテレーション順が不定のため、登録順ではなくキー長の降順（長い語優先）で適用してオーバーラップ時の取りこぼしを防ぐ。

これは ADR-0045 で不採用とした静的辞書を、「文脈不要な既知固有名詞の確実な読み」に限定して、LLM 方式を**置き換えるのではなく補完する**前段として再導入する判断更新にあたる。

## 結果

- 既知の固有名詞を LLM に依存せず確実に読ませられ、LLM の変換負荷も軽減される。
- LLM 方式（かな化・発音校正）はそのまま残るため、辞書未登録語や文脈依存の読みは従来どおり LLM が処理する。辞書未登録語は素通しされ変換されない。
- 静的辞書のため文脈依存の読み（同一表記で複数の読み）には対応しない。これは「1 つの読みに統一」という割り切りで許容する。
- LLM 呼び出しは増えない（既存パイプライン内の決定的な前処理として追加）。
- 設定はトップレベル `pronunciation` に置く。読み方は音声合成に関わるが、特定の話者・プリセットに紐づかない番組共通の語彙辞書であるため、`voicevox` 配下にネストせず独立セクションとした。

## 検討した代替案

- **`voicevox.pronunciation` へネスト**: 音声合成関連としては自然だが、辞書は話者・接続設定と独立した語彙テーブルであり、トップレベルの方が責務が明確。不採用。
- **LLM 変換マップの後段で辞書置換**: LLM のかな化結果を辞書で上書きする形になり、LLM の文脈判断を辞書が壊しうる。確実に読ませたい既知語を「先に」確定させる目的に反するため不採用（前段適用を採用）。
- **辞書のみで LLM 方式を置換**: ADR-0045 のとおり静的辞書は文脈依存に対応できず、未登録語の救済もできない。LLM 方式を残す補完に留める。
