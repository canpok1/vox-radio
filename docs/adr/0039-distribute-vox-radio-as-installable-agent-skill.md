# 0039. vox-radio をインストール可能なエージェントスキルとして配布する

- ステータス: 採用
- 日付: 2026-06-06

## コンテキスト

vox-radio の設定は複数ファイル（共通設定・エピソード仕様・アセット・フィード・Slack）に分かれ、フィールドが多くファイル間にキャラID参照などの整合性制約がある。利用者が一から書く／不備を直す負担が大きいため、設定の作成・修正をコーディングエージェントに任せたい。

そのためにはエージェントが設定リファレンスを手元で参照できる必要がある。本リポジトリの README をリモート参照させる方式は、ネットワーク前提・バージョン不一致（利用中バイナリと乖離）・README 肥大化の問題を抱える。

## 決定

`vox-radio install --skills` サブコマンドを新設し、利用者のプロジェクト配下 `.claude/skills/vox-radio/` に `SKILL.md` と `references/*.md`（設定ファイルごとに分割）をコピーする。

スキルの実体（`SKILL.md` と references）は `internal/cli/skills/vox-radio/` 配下で一元管理し `go:embed` で同梱する。install は同梱物をそのまま書き出すだけとし、エージェントはローカル参照のみで設定作成・更新が行える（利用中バイナリと常に一致）。本リポジトリは `.claude/skills/vox-radio` から当該ディレクトリへシンボリックリンクを張り、配布物と同じ実体でスキルを動作確認（ドッグフーディング）する。

リファレンスの正（source of truth）は、肥大化した README から設定リファレンス節を別ファイルに分離し上記ディレクトリへ置いたものとし、README はそこへリンクする。

## 結果

- 利用者は1コマンドでスキルを導入でき、ネットワーク不要・バージョン整合の取れたリファレンスで設定操作を任せられる。
- 配布物と本リポジトリの動作確認物が同一実体になり、両者の乖離が起きない。README も軽量化し管理点が1か所に集約される。
- リファレンス本文と `internal/config` 構造体の同期は引き続き手作業で、ドリフト監視（テスト等）が別途必要。
- 開発用シンボリックリンクは OS（Windows 等）によっては機能しないが、配布物は実ファイルのため利用者には影響しない。
- 既存の `templates/*.yaml`・`prompts/*.md`（ADR-0023）と同じ `go:embed` 配布に揃い、保守の一貫性が保たれる。

## 検討した代替案

- **README 参照型スキル（コピーしない）**: README を参照させる案。ネットワーク前提・バージョン不一致・README 肥大化が解消されず却下。
- **config.go から自動生成**: ドリフトは原理的に消えるが生成器の実装コストが高く過剰。将来の改善余地として保留。
