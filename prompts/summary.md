# [D] 番組要約プロンプト

以下のラジオ番組の話者付きセリフ一覧を読み、この回の番組要約と会話メモを生成してください。

## 入力

各要素は `{"speaker": "<speaker_role>", "text": "<セリフ本文>"}` の形式です。

```json
{{script_lines}}
```

## 出力形式

以下のJSON形式で回答してください。

```json
{
  "summary": "番組内容の要約（100〜300文字程度）",
  "conversation_notes": [
    {"category": "近況", "character_ids": ["zundamon"], "note": "新しくできたカフェにハマっている"},
    {"category": "掛け合い", "character_ids": ["zundamon", "metan"], "note": "めたんが食レポをからかい二人で盛り上がった"},
    {"category": "感想", "character_ids": ["metan"], "note": "新機能は便利そうと前向きに評価"},
    {"category": "ハプニング", "character_ids": ["zundamon"], "note": "言い間違えて言い直す一幕があった"}
  ]
}
```

## summary の注意事項

- 日本語で回答してください
- 今回の放送で「何を話したか」を具体的に記述してください
- 番組タイトルや固定的な説明文ではなく、この回固有の内容を書いてください
- リスナーがエピソードを選ぶ際の参考になる内容にしてください
- Show Notes として適切な文量（100〜300文字）にしてください

## conversation_notes の注意事項

- **rundown（記事の事実）に含まれない、会話から生まれた情報を幅広く拾う**こと。近況・掛け合い・感想・ハプニング・継続ネタなど種類は問わない
- 後からこの回の会話を思い出せる粒度で記述する
- `category` は内容に合う簡潔なラベルを自由に付けてよい（固定リストに縛られない）
- `character_ids` は入力の `speaker` に出てきたIDのみ使用し、存在しないIDを創作しない。特定キャラに紐づかない全体的な出来事は空配列にする
- 番組内で**実際に語られた・起きたことのみ**を書き、創作・推測を含めない
- 該当が無ければ `conversation_notes` は空配列にする
