# 作業メモ運用ルール

## ステップ完了時のチェックボックス更新

各ステップを完了したら、**その場で**作業メモのチェックボックスを `[x]` に更新すること。

- PR 作成・マージ・コミットなど節目となる操作が完了したら即時更新する（セッション終了時のまとめ更新は禁止）

## 作業ログ追記時の末尾確認と連番維持

作業ログにステップを追記するときは、以下の手順を守ること。

- 追記前に `Read` でファイル末尾を確認し、直前のステップ番号に続く連番で**ファイル末尾へ**追加する（途中挿入・順序逆転を防ぐ）
- チェックボックスを `Edit` で更新するときは、類似行が複数ある場合 `old_string` に十分な前後コンテキストを含めて一意性を確保してから実行する（二重更新を防ぐ）

## セッション終了前の一括更新（送り出し側）

セッションを終了する前に、以下を必ず完了してから終了すること。

- 完了した全チェックボックスを `[x]` に更新する
- 当セッションで実施済みだが作業ログに未記録の項目を追記する

さらに、以下のコマンドで git の実態を確認し、品質作業がコミット済みであることを確かめてから終了すること。

```bash
git log --oneline -5   # 直近コミット（完了済みステップ）を確認
git status             # 未コミット変更の有無を確認
```

確認対象の品質作業（以下がすべてコミット済み、または作業メモに「未実施・理由」付きで明記済みであること）:

- `/simplify`・self-review の対応コミット
- ADR（`docs/adr/`）の追加コミット
- `make docs` による `docs/cli/` の再生成差分

未コミットの品質作業が残っている場合は、コミットして push するか、作業メモに「未実施」と理由を明記してから終了する。

これにより、引き継ぎ側がチェックボックスと git 実態の乖離把握に費やす時間を削減できる。

## セッション引き継ぎ時の実状態確認

セッション引き継ぎ時は、作業メモのチェックボックスを鵜呑みにせず、以下のコマンドで git の実態（正）を確認してから作業を再開すること。

```bash
git log --oneline -5   # 直近コミット（完了済みステップ）を確認
git status             # 未コミット変更の有無を確認
```
