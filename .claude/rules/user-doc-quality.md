# 利用者向けドキュメント品質ルール

`README.md`・`docs/installation.md` など利用者向けドキュメントを変更したときは、利用者視点の品質を保つこと。

変更後の確認には `user-doc-reviewer` サブエージェントを呼び出し、下記の観点で指摘を洗い出す（read-only、修正はしない）。指摘は内容を吟味して反映する。

## 主な観点

- **冗長性** — 前置きフィラー文・文章説明・二重説明・重複・長文を避け、箇条書き/表に寄せる
- **利用者視点** — devcontainer・make・Docker 優先など開発者向けの概念/手段を利用者向けに混ぜない・先に出さない
- **情報配置** — セクションの責務に合わない詳細はリファレンスへ分離する
- **網羅性** — 列挙の漏れ（生成ファイル・設定・選択肢）に注意する
- **正確性** — コメントと実挙動の一致、古くなる値（バージョン等）に注意する
- **手順の順序** — 前提条件→操作の順にし、上から実行して失敗しないようにする
- **分かりやすさ** — 専門用語を説明し、ツールの役割を明示する。用語が初出の時点で意味を取れるようにする（前方参照を避ける）
