# 設定YAML・プロンプト編集ルール

対象ファイル: `*.yaml` / `examples/**/*.yaml` / `internal/cli/prompts/*.md` / `internal/config/testdata/*.yaml`

## 設定YAMLの編集

設定YAMLを編集するときは、以下を確認すること。

- **デフォルト値はコード定数から確認する** — Issue本文や設計ドキュメントの記載よりGoコードを正とする。`grep -r 'Default' internal/` で `const DefaultXxx` を確認してから記入する。
- **設定フィールドを追加・変更したらコード配線を確認する** — フィールド名を `grep -rn` で `internal/` 全体から検索し、宣言だけで未使用（サイレント無効化）になっていないか検証する。

## プロンプトファイルの編集

`prompts/*.md` を編集するときは、以下を確認すること。

- **出力JSONスキーマは `internal/model/*.go` と照合する** — プロンプトに記載するJSONフィールドは `internal/model/*.go` の `json` タグと1フィールドずつ照合し、設計ドキュメントのサンプルではなくGoの構造体定義を正とする。
- **セリフ生成プロンプト（`write.md`）に読み間違い対策ルールを含める** — VOICEVOX の誤読を防ぐため、以下の 3 点を「## 注意事項」に必ず含めること。
  1. 読み間違えやすい漢字（二人称・難読語・複数の読みを持つ語）はひらがな/カタカナで書く（例: 貴女→あなた）
  2. 英単語・英語ファイル名は発音を表す日本語に置き換える。拡張子も省略しない（例: `README.md`→リードミードットエムディー、`merge`→マージ）
  3. ファイルパス・URL・UUID 等の長い文字列はセリフに含めない

## テストデータの編集

テストデータを編集するときは、以下を確認すること。

- **必須フィールドを明示的に設定する** — 必須フィールドを省略すると後でテスト失敗になるため、対象構造体の全フィールドを明示的に含めること。
