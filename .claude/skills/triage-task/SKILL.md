タスクの登録先・ラベル運用は `.claude/rules/task-management.md` に従う。

## このスキルの位置づけ

タスクは作成時にラベルを付けない運用のため、**ラベルなし = 未トリアージのバックログ**である。本スキルはその1件を取り上げ「やる／やらない」を仕分けし、迷う点を詳細化する。

- `discuss`（タスクを作る）→ **`triage-task`（やるか決める・詳細化する）** → `assign-tasks`（割り当て）→ `solve-task`（やる）という流れに位置する。
- 本スキルは対象機能のコード実装には着手しない（仕分けと詳細化まで）。

## 手順

### ステップ1: 対象タスクの選択

- 引数 `$ARGUMENTS` に task-id 指定がある場合は、そのタスクを対象にする（`mcp__todoist__fetch-object`）。
- 未指定の場合は、`mcp__todoist__find-tasks` で `vox-radio` セクションの未完了タスクを取得し、**ラベルが1つも付いていない**ものだけを抽出する（`find-tasks` には「ラベルなし」の肯定フィルタが無いため、取得結果から各タスクの `labels` が空のものを選別する）。そのうち**作成が古い順**で1件を選ぶ（`ready` / `assign-to-claude` / `in-progress` など何らかのラベルが付くタスクは対象外）。
- ラベルなしのタスクが0件の場合は「未トリアージのタスクなし」と報告して終了する。
- 取得結果が空・想定外の場合は、次へ進む前に検索条件（`sectionId`）を見直して再取得する。

### ステップ2: タスク内容の確認

- `read-task` スキルを活用し、タイトル・本文・コメント・ラベルを把握する。

### ステップ3: コードベース調査

- 実施可否の判断に必要な範囲で、関係する構造体・設定・配線・既存タスク / ADR を調べる。
- 広域探索は `Agent`（`Explore` / `general-purpose`）を**並列**起動して結論のみ回収する。

### ステップ4: 実施可否の検討と詳細化

- 「やるべきか」を、価値・重複（既存タスクとの重なり）・実現性・前提崩れの観点で分析する。
- **担当者が判断に迷う曖昧点を漏れなく洗い出す**（仕様の解釈余地、対象範囲の境界、データ形式、エラー時挙動、命名・配置、依存追加の要否、後方互換性など）。
  - 技術詳細（命名・配置・nil初期化方針など）は既存規約・パターンに沿って**スキル側がデフォルトを決め**、タスク本文を具体化する（担当者の判断を残さない）。
  - 製品判断（公開/非公開、対象範囲、コスト増を伴う方式選択など）はステップ5でユーザーに確認する。

### ステップ5: ユーザーへ実施可否を確認

- `AskUserQuestion` で実施可否を確認する（推奨案を第1選択肢に「(推奨)」付きで提示）。
- 判断材料として、ステップ4の分析結果と詳細化案を添える。

### ステップ6: 結論に応じた処理

- **実施しない**場合 → 破棄する。
  1. 破棄理由（なぜ実施しないか）を `mcp__todoist__add-comments` でタスクにコメントする。
  2. その後 `mcp__todoist__complete-tasks` で完了にする（記録を残すため `delete-object` は使わず完了扱いとする）。
  - 理由コメントを残す前に完了しないこと（後から判断根拠が追えなくなるのを防ぐ）。
- **実施する**場合 → ready 化する。
  1. ステップ4で詳細化した内容を `mcp__todoist__update-tasks` で `description` に反映する。
  2. `ready` ラベルを付与する（`update-tasks` の `labels` は全置換のため、取得済みの現ラベルをベースに `ready` を追加する）。

### ステップ7: 仕上げ確認

- 反映後のタスクを `mcp__todoist__find-tasks` / `mcp__todoist__fetch-object` で読み返し、本文の構成崩れ・ラベル付与漏れ・破棄漏れがないか確認する。

## 制約・設計上の注意

- **判断はユーザー、詳細はスキル**: 実施可否の最終判断は `AskUserQuestion` でユーザーに委ね、再現可能な技術詳細はスキルが既存規約から決めてタスクに固定する。
- **破棄は完了扱い**: `delete-object` は使わず `complete-tasks` を用いる（誤判断時の復元・トレースを残すため）。理由コメントは完了前に残す。
- **ラベルは全置換に注意**: `update-tasks` の `labels` は全置換のため、現ラベルを保持したまま追加/除去する（`.claude/rules/task-management.md` 準拠）。
- **タスク作成時のラベル不付与ルールとの整合**: 本スキルはタスクを「作成」しないため当該ルールには抵触しない。`ready` 付与は人手による着手可否判断の代替として、ユーザー確認を経て行う。
- **実装はしない**: CLAUDE.md「明示的な実装指示がない限りファイル編集・作成をしない」に従い、対象機能のコードには着手しない。
