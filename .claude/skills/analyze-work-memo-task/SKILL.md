タスクの登録先・ラベル運用・本文の書き方は `.claude/rules/task-management.md` に従う。

## 手順

- [ ] ステップ1: アーカイブ済みメモを収集する
- [ ] ステップ2: メモの内容を読み込み・分析する
- [ ] ステップ3: 改善点を抽出・整理する
- [ ] ステップ4: 改善タスクを作成する
- [ ] ステップ5: 分析済みメモをアーカイブ移動する
- [ ] ステップ6: 結果を報告する

## 注意点

- 全体
    - 分析は事実ベースで行うこと。推測や主観的な評価は根拠となるメモの記述を添えること。
    - 改善タスクは具体的かつ実行可能な内容にすること。抽象的な提案は避ける。
    - 既存の未完了タスクと重複するタスクを作成しないこと。ステップ4の前に `mcp__todoist__find-tasks` で確認する。
- ステップ1
    - アーカイブ済みの作業メモを収集すること。
    - アーカイブ済みメモが存在しない場合はその旨を報告して終了すること。
- ステップ2
    - 全てのアーカイブ済みメモを読み込むこと。
    - 各メモから以下の情報を抽出すること:
        - 作業の種類（機能追加、バグ修正、リファクタリングなど）
        - 遭遇した問題・つまずき
        - 振り返りのKeep/Problem/Try（存在する場合）
        - 作業の流れで非効率だった点
- ステップ3
    - 複数のメモを横断して、以下の観点で改善点を抽出すること:
        - 繰り返し発生している問題パターン
        - 複数のメモで共通して挙がっている改善提案（Try）
        - ワークフローやツールの非効率性
        - スキルやスクリプトの改善機会
    - 抽出した改善点をタスク単位にグルーピングすること。1つのタスクには1つの改善テーマを割り当てる。
    - 改善点が見つからない場合はその旨を報告して終了すること。
- ステップ4
    - 既存の未完了タスクを `mcp__todoist__find-tasks`（`vox-radio` セクション）で取得し、重複がないか確認すること。
    - 重複しない改善点についてのみ `mcp__todoist__add-tasks` でタスクを作成すること（`projectId` / `sectionId` は `.claude/rules/task-management.md` に従う）。
    - タスクには以下を含めること:
        - `content`（タイトル）: 改善内容を簡潔に示す
        - `description`（本文、Markdown）:
            - 背景（どのメモのどの記述から抽出したか）
            - 改善提案の具体的な内容
            - 期待される効果
        - Todoist タスクにフッターは付けない（フッターは GitHub の PR・コメント向け）。
- ステップ5
    - 分析対象としたメモファイルを分析済みディレクトリへ移動すること。
    - これにより、次回実行時に同じメモが再分析されることを防ぐ。
- ステップ6
    - 作成したタスクの一覧（ID・タイトル）を報告すること。
    - 改善点が見つからなかった場合や全て既存タスクと重複していた場合は、その旨を報告すること。
