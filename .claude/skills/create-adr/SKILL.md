## このスキルの起動タイミング

- **手動実行**: ユーザーが `create-adr` を明示的に呼んだとき。
- **LLM 判断による自動起動**: 会話の中で「記録すべき重要判断」を行った/確定したと判断したとき。

### 記録すべき重要判断（起動する）
- アーキテクチャ・技術選定の決定/変更（依存の追加・除去、ライブラリ/プロトコル選択、データ形式、配信方式 など）
- 後から覆すコストが高い、または「なぜそうしたか」が失われると困る判断
- 複数案を比較して1案に絞った判断

### 記録不要（起動しない＝ADR スパム防止）
- 些末な実装詳細・命名・整形
- 既存方針の単なる適用
- 容易に覆せる小さな選択

## 手順

1. **自動起動の場合は先に確認する**（手動実行時はスキップ可）
   - 「重要な判断のようです。ADR に記録しますか？」とユーザーに確認してから次へ進む。
   - CLAUDE.md の「無断でファイル作成をしない」方針に従い、いきなりファイルを作らない。

2. **次の連番を決定する**
   - `docs/adr/` が無ければ作成する。
   - 既存の `docs/adr/NNNN-*.md` から最大番号を求め、+1 した4桁ゼロ埋め番号を採番する。
     ```bash
     mkdir -p docs/adr
     last=$(ls docs/adr 2>/dev/null | grep -oE '^[0-9]{4}' | sort -n | tail -1)
     next=$(printf '%04d' $(( 10#${last:-0} + 1 )))
     echo "$next"
     ```

3. **内容を組み立てる（対話組み立て型）**
   - 会話の文脈・git 差分（`git diff` / `git log`）から「コンテキスト/決定/結果/検討した代替案」を抽出する。
   - 不足している情報は **1問ずつ** 質問して補う。

4. **ファイルを作成する**
   - ファイル名: `docs/adr/NNNN-<english-kebab-title>.md`（タイトルは英語ケバブケース）。
   - 本文は **日本語**、下記テンプレートに従う。
   - **フッター（🤖 Generated...）は付けない**（ADR は Issue/PR ではないため）。

5. **文字数をチェックする（冗長化防止）**
   - **散文（地の文）合計の目安は約 1,200 文字、ハード上限は約 1,350 文字**。
   - セクション別目安: コンテキスト ≤450 / 決定 ≤300 / 結果 ≤300 / 検討した代替案 ≤300。
   - **計測対象外**: コードブロック・表・見出し・ステータス/日付行。
   - 作成後に散文部分を計測し、超過していたら要点を残して圧縮し上限内に収める。
     ```bash
     # 目安: コードブロックを除いた本文の文字数を確認する
     wc -m docs/adr/NNNN-*.md
     ```

6. **インデックスを更新する**
   - `docs/adr/README.md` が無ければ見出し＋一覧表を作成する。
   - 一覧表に新しい ADR の行（番号・タイトル・ステータス・日付）を1行追記する。

7. **ドラフトをレビューしてもらう**
   - 作成した ADR とインデックス追記をユーザーに提示し、確認を得て確定する。
   - 修正依頼があれば反映してから再提示する。

## ADR テンプレート（MADR 軽量版・日本語・フッターなし）

```markdown
# NNNN. <タイトル>

- ステータス: 採用
- 日付: YYYY-MM-DD

## コンテキスト

<なぜこの判断が必要になったか。背景・制約>

## 決定

<何を選んだか>

## 結果

<良い影響・悪い影響・トレードオフ>

## 検討した代替案

<比較した他案と却下理由（各案を簡潔に）>
```

- ステータスの値: `提案` / `採用` / `却下` / `廃止` / `置換`（既定は `採用`）。
- 日付は実行日（`date +%F`）を使う。

## インデックス（docs/adr/README.md）の形式

```markdown
# Architecture Decision Records (ADR)

重要度の高い判断を記録する。フォーマットは MADR 軽量版（日本語本文）。

| 番号 | タイトル | ステータス | 日付 |
|------|----------|------------|------|
| [0001](0001-remove-vox-actor-dependency.md) | vox-actor 依存の除去 | 採用 | 2026-05-30 |
```
