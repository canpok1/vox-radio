## CLAUDE.mdの方針

- このファイルはできるだけシンプルに保つ
- 詳細なルールは `.claude/rules/` に分離する（パス固有ルールで必要時のみ読み込まれる）
- スキル固有の情報は `.claude/skills/` のスキル定義に記載する（スキル呼び出し時のみ読み込まれる）

## 実装の開始タイミング
- ユーザーから明示的に実装の指示がない限り、ファイルの編集・作成を行わないこと
- 調査・分析・Issue更新などの非実装タスク中に、勝手に実装を始めないこと
- 実装を開始したい場合は、まずユーザーに確認を取ること

## GitHub IssueとPRの管理
- IssueやPR、コメントを作成する際は以下を守ること
  - 末尾にフッターを付与する
    ```
    ---
    🤖 Generated with [Claude Code](https://claude.ai/claude-code)
    ```
  - 作成内容に不備がないか作成後に確認すること（例: Issueのレイアウトが崩れている、フッターが付与されていないなど）
