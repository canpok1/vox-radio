# [C] flow設計プロンプト

以下の番組構成全体と対象コーナーの情報を元に、対象コーナーの話の流れ（flow）を設計してください。

## 対象コーナー

`appearance_count` は今回を含むこのコーナーの扱い回数（`1` = 今回が初めて = 新コーナー）。
`last_episode_number` は前回このコーナーを扱った回番号（省略時は過去に扱いなし）。

## 番組構成全体（全コーナーの title・summary・選別理由・キャスト一覧）

`casts[].appearance_count` は過去の出演エピソード数（今回含まず）。`0` = 今回が初登場。`casts[].last_episode_number` は前回出演した回番号（省略時は過去に出演なし）。
各コーナーの `appearance_count` は今回を含む扱い回数（`1` = 新コーナー）、`last_episode_number` は前回扱った回番号（省略時は過去に扱いなし）。

```json
{{program}}
```

## 出力形式

以下のJSON形式で回答してください。

```json
{
  "flow": "記事1でAIチップの性能向上を紹介し、次に記事2でその普及課題を取り上げる。前のコーナーの話題を受けて自然につなぎ、次のコーナーへ渡す"
}
```

## 注意事項

- **番組構成全体と対象コーナーの選別理由を踏まえて flow を設計し、他コーナーと矛盾・重複しないようにすること**
- **位置別の役割**:
  - `opening`（先頭・導入）: 番組全体の導入として、これから扱う話題・コーナーの流れを踏まえリスナーを引き込むこと。各コーナーの詳細紹介・解説は当該コーナーに委ね先取りしない。番組を締めない。
  - `ending`（末尾・締め）: 番組で扱った内容を踏まえて自然に締めること。
  - `middle`（中間・つなぎ）: 前後のコーナーを自然につなぐこと。
- **記事ありコーナー**: 選別済み記事と選別理由に沿って紹介順・繋ぎを設計すること
- **記事なしコーナー**: 記事がなくても番組構成全体に沿って矛盾しない flow を設計すること
- **全コーナーに記事がない回**: その構成を前提に矛盾しない flow にすること
- `flow` は台本を書く際の指針となるよう、具体的な構成・繋ぎ方を記述してください
- 特定コーナー名（ニュース、エンディング等）をハードコードせず、番組構成から読み取った内容に基づいて設計すること
- 日本語で回答してください
