# セキュリティポリシー

## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、**公開 Issue や Pull Request では報告しないでください**。情報が公開されると、修正前に悪用されるおそれがあります。

代わりに、GitHub の **Private vulnerability reporting**（非公開の脆弱性報告）機能を利用してください。

1. リポジトリの [Security タブ](https://github.com/canpok1/vox-radio/security) を開く
2. **「Report a vulnerability」** から非公開で報告する

報告には以下を含めていただけると調査がスムーズです。

- 脆弱性の概要と影響範囲
- 再現手順（可能であれば PoC）
- 対象のバージョン・環境

## 対応方針

- 報告を受け取り次第、内容を確認し、修正の要否・優先度を判断します。
- 個人開発のため対応に時間がかかる場合があります。あらかじめご了承ください。
- 修正後、必要に応じて GitHub Security Advisory として公開します。

## サポート対象バージョン

原則として最新リリースのみをサポート対象とします。
